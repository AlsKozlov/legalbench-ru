## Зачем это нужно

Юристов по всему миру [штрафуют за фейковые ссылки от ИИ](https://www.legaldive.com/news/chatgpt-fake-legal-cases-generative-ai-hallucinations/651557/),
а Stanford RegLab [показал](https://arxiv.org/pdf/2405.20362), что даже платные
Lexis+AI (17%) и Westlaw (33%) галлюцинируют. Для **русского права и русских
моделей** этого никто открыто не мерил — потому что открытого бенчмарка не было.

Теперь есть. И местами хуже западного: на нашем самом честном срезе русские модели
**галлюцинируют ссылки на постановления судов в 35–48% случаев**.

## 🏆 Baseline (closed-book, весь бенч, предварительно)

| модель | overall | knowledge | reasoning | tool-use |
|---|---|---|---|---|
| **DeepSeek-v32** | **76%** | 53% | **91%** | 85% |
| **YandexGPT 5.1** | **76%** | **62%** | 85% | 82% |
| YandexGPT-5 Pro | 72% | 52% | 83% | 85% |
| Qwen3-235B | 71% | 48% | 85% | 84% |
| Qwen3.6 | 65% | 42% | 76% | 81% |
| GPT-OSS-120B | 57% | **21%** | 76% | 80% |

**Узкое место у всех — знание права (21–62%), а не рассуждение (76–91%).**
GPT-OSS знает право РФ втрое хуже YandexGPT, но рассуждает почти как все → чините
знание (RU-tuning / RAG), а не логику.

## 🎯 Что меряем (4 трека)

| Трек | Кейсов | Что проверяет |
|---|---|---|
| **knowledge** | 311 | память: какая статья/норма регулирует институт (citation, rule-recall) |
| **reasoning** | 348 | вывод по фактам, толкование, расчёты, применение позиций судов |
| **tool-use** | 187 | выбор правильного инструмента к госреестру + аргументы (BFCL-RU-стиль) |
| **temporal** | в reasoning | устойчивость к устаревшей редакции закона |

**7 типов рассуждения** (по мотивам [LegalBench](https://arxiv.org/abs/2308.11462)):
rule-recall, rule-application, rule-conclusion, issue-spotting, interpretation,
rhetorical + RU-специфичный **citation** (точность ссылки на норму/постановление).

**5 форматов ответа** → **детерминированный скоринг** (без LLM-судьи, воспроизводимо):

| формат | пример | скоринг |
|---|---|---|
| `binary` | Да / Нет | exact-match извлечённой метки |
| `multiple_choice` | A / B / C / D | exact-match буквы |
| `extraction` | «три года», «500 000» | нормализация + accept-варианты |
| `norm_citation` | `ГК РФ ст.333` | **norm-F1** (парсинг (кодекс, статья)) |
| `tool_call` | `efrsb.check_company_bankruptcy(inn=…)` | **AST-match** (tool + args) |

### ⭐ Главная фишка — условия прогона (`--mode`)

Один и тот же кейс гоняется при **разном поданном контексте** — это превращает
викторину в каузальный эксперимент:

| `--mode` | в промпте | что измеряет |
|---|---|---|
| `closed` | только вопрос | знание (память) |
| `grounded` | + верная норма | рассуждение при идеальном ретриве |
| `distractor` | + **неверная** норма | поведётся ли на ложь (counterfactual robustness) |
| `temporal` | + **устаревшая** редакция | анкорится ли на старый закон |
| `reject` | нормы нет | умеет ли отказаться (negative rejection) |

Ценность — в **дельтах**: `closed→grounded` (упор в память ⇒ нужен RAG),
`grounded→distractor` (ведётся ли на ложную норму — safety), temporal-flip.

## 🚀 Быстрый старт

```bash
# проверка скорера (должно быть 100% — эталонные ответы из gold)
python score.py --model oracle

# своя модель через любой OpenAI-совместимый endpoint
export RU_EVAL_BASE_URL=https://...  RU_EVAL_API_KEY=...
python score.py --model openai --model-name <модель>

# по дорожке и в режиме grounded (рассуждение при поданной норме)
python score.py --model openai --model-name <модель> --mode grounded --track reasoning

# валидация датасета + QA-аудит + матрица покрытия
python validate.py && python qa.py && python coverage.py
```

Зависимостей нет — только стандартная библиотека Python 3.11+.

## ✅ Как обеспечено качество

1. **Адверсариальная верификация при генерации** — 2–3 независимых агента-скептика
   на каждый кейс, выживают только единогласно прошедшие.
2. **QA-линт** ([qa.py](qa.py)) — ловит геймабельность (перекос букв в MC), утечки
   ответа в вопрос, дубли.
3. **Red-team аудит** — 26 скептиков на выборке 154 кейса
   пытались опровергнуть gold. **Измеренный error-rate: 3%.** Поймал и исправил
   реальные баги, включая галлюцинации норм у самого генератора.
4. **Контроль скорера** — `oracle` = 100%, `stub` = 0%.

## 🔭 Куда растём

**v0.4** — уголовный + семейный треки, полные условия на весь корпус, трек свежих
НПА (anti-contamination) · **v0.5** — юр-ревью → снятие `needs_expert_review` →
публикуемые baseline · **v1.0** — ≥1500 кейсов, длинные документы, arXiv + MERA.

Бенч — не только лидерборд, а **сигнал награды для дообучения**: парсер норм общий
с тренировочным пайплайном, а citation-verifier работает и как фильтр данных, и как
reward для RLVR. *RAG — для фактов, fine-tuning — для поведения, RLVR — для цитат.*

## ⚠️ Честные оговорки

- **Юр-ревью живым юристом не пройдено** → цифры предварительные, всё
  `needs_expert_review`. Red-team — это LLM-скептики (коррелированные слепые зоны).
- Не покрыты **уголовное и семейное** право.
- Нет длинных документов и генерации (драфтинга) — меряем дискретный вывод.
- Это **release candidate**, не финальный эталон. И мы это не прячем.
